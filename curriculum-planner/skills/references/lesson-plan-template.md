# Lesson Plan Template & Quality Checklist

This document defines the output template for lesson plans, provides a worked example, and includes the pre-delivery quality checklist.

---

## Worked Example: Activity Block

Here is a fully worked example of one activity block from Session 1:

```
┌─────────────────────────────────────────────────────┐
│ 10:15–10:45  First Prompt, No Instructions  (30 min)│
│ Individual → Pairs → Full Group                      │
│ GRR: You Do → You Do Together → We Do                │
├─────────────────────────────────────────────────────┤
│                                                      │
│ TEACHER DOES                                         │
│ • Project one prompt on screen: "Write me a          │
│   professional email declining a meeting politely."   │
│ • Give no other instructions. Let students struggle.  │
│   This is intentional — students need to feel the     │
│   gap before CRAFT will land.                         │
│ • Circulate silently. Note who asks questions vs.     │
│   who dives in. Note the range of output quality.     │
│ • After pair comparison, cold-call 3-4 pairs to       │
│   share what they noticed.                            │
│ • Highlight patterns: "Notice how some of you got     │
│   formal, some got casual? That's because you         │
│   didn't specify..."                                  │
│                                                      │
│ STUDENTS DO                                          │
│ • Open Claude, enter the prompt exactly as shown      │
│ • Read their own output silently (2 min)              │
│ • Pair up and compare outputs — what's different?     │
│ • Share observations with class when called on        │
│                                                      │
│ ⏱ PACING                                            │
│ Prompt + generate = 5 min. Read own output = 2 min.  │
│ Pair up and compare = 5 min. 3-4 pairs share what    │
│ they noticed = 8 min. Instructor highlights           │
│ patterns = 5 min. Buffer/transition = 5 min.          │
│                                                      │
│ 💡 TEACHER TIP                                       │
│ Resist the urge to teach during this block. The       │
│ point is to let students discover the problem.        │
│ Teaching happens next.                                │
│                                                      │
│ ✓ CHECK: Are students noticing the gap between        │
│   what they got and what they wanted? If <25% can     │
│   articulate the gap, show two contrasting outputs    │
│   side-by-side before moving to CRAFT.                │
└─────────────────────────────────────────────────────┘
```

**Why this works:**
- Teacher actions are specific and sequenced, not "teacher facilitates discussion"
- Student actions describe what students actually do, not "students participate"
- Quoted teacher scripting gives exact language to use
- Pacing breaks the 30 minutes into accountable chunks
- The teacher tip addresses a common instructional mistake
- The CFU check tests the actual learning target, not just engagement

## Pre-Delivery Quality Checklist

Run this checklist before marking any lesson plan as complete. Every item must pass.

### Objectives
- [ ] Every SWBAT uses an observable, measurable verb
- [ ] Every SWBAT is tagged with a Bloom's level
- [ ] No SWBAT uses "understand," "learn about," or "appreciate"

### Assessment Alignment
- [ ] Every SWBAT has at least one formative assessment point in the lesson flow
- [ ] Exit ticket questions map to specific SWBATs
- [ ] Assessment methods match Bloom's level (don't use recall for an "Evaluate" SWBAT)

### Lesson Flow
- [ ] Every instructional block has a GRR phase label
- [ ] No block jumps from I Do to You Do Alone
- [ ] Teacher actions are specific (include suggested language, not just "teacher explains")
- [ ] Student actions describe what students produce or practice (not "students listen")
- [ ] Timing adds up correctly (block durations match start/end times)

### CFU Checkpoints
- [ ] CFU every 10-15 minutes of instruction
- [ ] Each CFU specifies the technique (not just "check for understanding")
- [ ] Each CFU includes what to do if students don't get it

### Differentiation
- [ ] Scaffold-down notes for struggling learners
- [ ] Scaffold-up notes for advanced learners
- [ ] Language support notes for multilingual learners

### Formatting
- [ ] Visual hierarchy is clear — headers, labels, and body text are visually distinct
- [ ] Delivery text is not truncated — full instructions are included
- [ ] Break blocks are visually separated from instructional blocks
- [ ] Document is scannable — a teacher can find the current block within 3 seconds
